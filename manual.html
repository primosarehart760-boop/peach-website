Section 01
快速开始

三步启动，最快 2 分钟出产品方案 + Demo。

1

打开专属 Chrome

双击桌面的 启动桥.command，它会自动启动本地服务并打开绕过代理的专属浏览器窗口。不能用普通 Chrome，Clash 梯子会拦截本地请求。

2

输入你的产品需求

在首页大输入框里写一句话，比如"宠物医疗 AI 问诊平台"或"给中小学生的 AI 写作辅导工具"。选行业（可选），选模式（闪电档 or 标准档）。

3

点击启动，等待交付

闪电档约 2 分钟出 3 份文档 + Demo；标准档 20-40 分钟自动跑完 P1→P6 全链路，包含市场、用户、产品、设计、技术、运营 共 30+ 份专业文档。

Section 02

Section 03
两档模式

根据你的时间和需求深度，选择合适的档位。

⚡

闪电档

2 分钟出 3 份核心文档 + 1 个可点击 Demo。适合快速验证方向、对外汇报、客户演示用。

产出：Product Brief · Design-Tech Brief · Demo HTML

约 2 分钟
🏭

标准档

P1→P6 全链路自动流水线，含 CEO Gate 质量关口，30+ 份大厂标准文档。

产出：市场报告 · 用户研究 · PRD · 技术方案 · 运营计划 · Demo 等

20–40 分钟
推荐做法：先跑闪电档验证方向 → 方向对了再跑标准档的前端阶段（P3+P4）→ 前端确认后再跑后端阶段（P5+P6）。标准档可以只勾选你需要的阶段，不必每次全跑。

P1

市场洞察

市场数据报告、竞品深度分析、AI 边界分析（N34）、竞争壁垒报告、SWOT + 商业计划书、BRD + MRD。并发运行，约 8–12 分钟。

P2

用户研究

用户调研报告、用户画像、用户旅程地图、产品定位 Charter、用户问卷设计。P1 全部完成后自动启动。

🚦

CEO Gate · 质量关口

P2 完成后 AI CEO 自动对 5 个维度打分（满分 500）。低于 400 回炉重做，低于 300 终止项目，连续 2 次不过强制推进。通过后 P3–P6 才会解锁。

P3

产品规划

PRD 产品需求文档、产品路线图、功能清单、MVP 方案、信息架构 + 数据指标体系。

P4

设计体验

交互原型文档、UI 设计规范、Demo HTML（带素材和初稿）、AI 模型选型报告、Design QA 审查。

P5

技术开发

技术方案报告、API 接口文档、数据库设计、Embedding 测试方案、AI 评测报告、SOP 技术安全规范。

P6

上线运营

上线运营计划、ASO + 增长数据复盘、合规隐私文档、项目归档 + 变更日志。

QA

Reviewer + CKO 蒸馏

全链路一致性审查（PRD ↔ Demo ↔ 技术方案三向对齐）。CKO 将本次项目的 Prior Art 和 Anti-Pattern 写入知识库，下次项目自动学习。

Section 05
CEO Gate · 质量关口

P2 完成后，AI CEO 自动阅读全部 P1+P2 文档，对 5 个核心维度打分，决定项目是否值得进入深度生产。

Q1 · 市场空间
有没有量化的 TAM/SAM 数字和增速？市场时机对不对？
Q2 · 竞品壁垒
差异化切入点够不够具体？能不能打穿竞品核心护城河？
Q3 · AI 边界
哪些功能必须 AI、哪些用传统方案更好？边界分析是否完整？
Q4 · 用户对齐
用户画像够不够具体？痛点有没有数据或访谈佐证？
Q5 · 产品定位
产品定位能不能一句话说清楚？MVP 范围够不够收敛？北极星指标是否量化？
总分判决后续动作
≥ 400 且无低分项 PASS 解锁 P3–P6，全速推进
300–399 或有低分项 REDO 指定 Agent 返工，重新评分（最多 2 次）
< 300 或多个低分项 TERMINATE 市场机会不足，建议终止

Section 07
一人公司哲学

桃子公司不是一个内容生成工具，而是一个可调度、可审查、可迭代的 AI 产品公司系统。

🔀

可调度

任务自动路由 → 调对应员工 → 并发 + 串行有规则，不需要你手动安排谁做什么。

🛡

可审查

每一步都有硬审查（CEO Gate + QA + Reviewer），全程可回溯，产出有依据不是拍脑袋。

🔁

可迭代

每跑一次更聪明。CKO 把成功案例和失败教训写入知识库，下次自动学习。

你的角色：老板。提需求、看工作台、最终拍板。AI 员工负责执行，不需要你懂代码或写文档。

核心等式：马海倩（1人）+ 有人格的 AI 工作流（42员工）= 10 人大厂团队的产出质量。

三大信条：Demo 是文档的可视化表达，不是为了 Demo 做 Demo；员工是老板方法论的外化，不是通用工具人；老板有最终否决权，AI 给数据给选项，但你说了算。

Section 08
壁垒不在模型，在系统

大模型每隔几个月就升级一次，这不是威胁，是底层基础设施的免费升级。真正的护城河，不在你用哪个模型，在你建了一套什么样的系统。

「别人用锤子打钉子，我们建了一座工厂」
竞品换了个更强的模型，等于买了把新锤子。桃子换了个更强的模型，42个Agent同时升级——工厂效率整体跃升。
模型迭代越快，系统型产品的优势越大。
⚙️
工作流编排
42个Agent的调度逻辑、并发规则、8层审查链——这套协作机制是竞品抄不走的肌肉记忆。模型换了，编排层不动。
📚
方法论外化
VELA 33份模板 + 桃子PRD 15板块，是马海倩6年产品经验被结构化编码进系统的成果。这不是Prompt，是产品判断力的固化。
🧬
知识库积累（越跑越聪明）
每次跑完，CKO把成功案例写入Prior Art、失败教训写入Anti-Pattern。下次同类项目自动调用。跑100次，就有了100个项目的经验沉淀。
🌐
行业深度专家池
200位行业专家Agent，每位都掌握该行业的机会点、弊端、获客路径、迭代打法——这是竞品靠换模型追不上的行业洞察深度。
面对模型升级，我们怎么打
1

模型是可替换的底层，上层工作流不动

Claude 4 → GPT-5 → Qwen3，只需换 API key。42个Agent的角色、方法论、审查逻辑、VELA模板全部保留。升级成本接近零，竞争力整体跃升。

2

竞品跟进了功能？我们的差异在产出质量标准

工具层（"也能生成PRD"）很容易被追平，但质量层（8层审查 + N34 AI边界分析 + 反指标体系）不是加个功能就有的。我们的产出是有依据的，不是拍脑袋的。

3

越用越强，形成数据飞轮

每跑一个项目，知识库更丰富；知识库越丰富，下一个项目产出质量越高；质量越高，信任度越高，使用频率越高。这个飞轮，新进入者没有。

结论：模型快速迭代，对桃子是顺风，不是逆风。我们的竞争优势来自系统深度，不来自模型版本号。

Section 09
常见问题
Q

网页打不开 / 502 报错？

必须用"启动桥.command"打开的专属 Chrome，不能用普通浏览器。Clash 梯子会拦截 localhost 请求。

Q

跑到一半停了怎么办？

进"📁 项目历史"页面找到这个项目，展开查看哪个 Agent 没跑完，点"迭代更新"重新触发。

Q

方向变了，已有项目怎么更新？

去"📁 项目历史"，找到对应项目，在"迭代更新"框里写你改变了什么（比如"目标用户从 B 端改为 C 端"），勾选受影响的 Agent，提交。系统会先给旧文档做版本快照，再重跑选中的 Agent。

Q

CEO Gate 打了 TERMINATE，还能继续吗？

可以。连续 2 次 REDO 未通过时系统会强制推进，或者你可以去项目历史手动触发迭代，补充 CEO 指出的缺失信息后重跑 P1/P2 Agent。

Q

想只出某一份文档，不跑全套？

标准档里勾选对应阶段即可。比如只要 PRD，勾 P3；只要技术方案，勾 P5。已跑完的阶段不会重复计费。
