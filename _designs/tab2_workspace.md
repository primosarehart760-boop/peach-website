# Tab 2 · 分析工作台（workspace.html）· 多角色构思

> **定位**：第一层分析引擎的落点。用户从 index 提交需求后 · 进入这里看 CEO 跑六步分析 · 生成 10 件套的雏形 · 每段有"依据感"标注。
> **对应盲区**：PDF §15.2 分析引擎 · PDF §8.2 没有第一层分析引擎

---

## 👑 CEO 定位

- 这是桃子公司的**生产车间**· 用户看着桃子系统怎么把需求拆成方案初稿
- 输入：URL 参数 `?q={需求}&industry={行业}&mode={模式}` 或 sessionStorage 里的 `peach-req`
- 输出：
  - 闪电档（默认）：3 分钟内跑完六步分析 · 生成 3 份文档 + 1 demo
  - 标准档：展示 10 件套进度 · 每件套实时标"生成中 / 已完成 / 待审"

- **核心体验**：用户看得到系统在调什么资源（"基于 VELA 11 PRD 模板 / 蔡雨洋训练师笔记 / 医疗行业专家框架"）· 反"像在瞎编"
- 60 秒内必须让用户看到**第一段产出**（用户画像）· 不让用户干等

## 🧑‍🎤 PM 视角 · 交互逻辑表 5 态

| 状态 | 触发 | 系统行为 |
|---|---|---|
| **正常** | 从 index 跳入带完整参数 | 渲染需求卡 + 顺序跑 6 步分析 |
| **空状态** | 直接访问 workspace.html 无参数 | 显示引导卡 "请先从首页输入需求" + 跳 index 按钮 |
| **加载中** | 每一步分析生成时 | Skeleton 占位 · 进度条动 |
| **首次使用** | localStorage 无 `peach-visited-workspace` | 顶部浮条 "这是分析工作台 · 会自动跑 6 步" |
| **禁用态** | 参数损坏 / 解析失败 | 红字 "参数异常 · 请重新从首页开始" + 回 index 按钮 |

## 🧑‍🎤 PM 视角 · 边界异常矩阵 6 类

| 类型 | 处理 |
|---|---|
| 网络断开 | 纯前端生成 · 不依赖网络 · 一切模拟数据 |
| 数据为空 | 参数缺 → 显示引导回首页 |
| 并发冲突 | 不涉及 |
| 权限不足 | 不涉及 |
| 输入超限 | index 已拦 · 到这里 q <= 500 字 |
| 服务端错误 | 不涉及（纯静态） |

每个 step 卡可**展开**显示生成的内容：
- 用户画像：姓名/年龄/职业/一日场景（JTBD 风格）
- 产品定位：一句话定位公式 + "不是什么"清单
- 市场竞品：TAM/SAM/SOM + 3 家竞品 × 5 维
- 北极星：指标 + 3 个反指标（护栏）
- 差异化：3 条差异 + 3 层壁垒
- PRD 骨架：15 板块标题 + 阶段标注

### 闪电档 vs 标准档显示差异
- **闪电档**：6 步全跑 · 每步 1-2 句精简 · 2 分钟内全出完 · 自动生成 3 份固定文档
- **标准档**：6 步全跑 · 每步深度展开 · 30 分钟内 · 并展示 10 件套进度条

## 🎭 前端视角 · 实现

### 技术栈
- 纯 HTML + CSS + 原生 JS
- 共享 `css/main.css` · 新增 `<style>` 内联
- 分析内容：**基于行业 + 模式的静态模板**（不接真 AI）· 留钩子未来接 Claude Agent SDK

### 行业模板映射
每个行业一个 JS 对象 · 包含 6 步的模板数据：
```js
const INDUSTRY_TEMPLATES = {
  金融: {
    画像: '林建国 · 38 岁 · 消费金融信贷审核员 · 日均审核 80 单 · 疲劳度高',
    定位: '为银行信贷审核员提供 AI 辅助审核系统 · 把 45 分钟/单 压到 4 分钟',
    // ...
  },
  医疗: { ... },
  ...20 行业
};
```

### 分析流跑动
- 页面加载后 · 读参数 + 模板
- 用 setTimeout 模拟"思考时间"（闪电档每步 10-15 秒 · 标准档每步 180 秒）
- 每步完成后揭示内容 + 依据感栏
- 全部完成后底部"下一步"按钮激活

### 暂停 / 跳过 / 重跑
- 顶部控制条给用户主动权
- 暂停：停下当前 step
- 跳过：跳过本 step · 后面继续
- 重跑：清空当前 step 数据重新生成

## 🧪 QA 审查点（对照 5 字总纲）

### 约束
- URL 参数缺 → 拦截 · 引导回首页
- 时间闸门：闪电 2 分钟超时 · 标准 30 分钟超时 · 用大红字提醒

### 审核
- 不用真实面过的公司名 · 只用虚构通用（某金融机构 / 某教育平台）
- 不出现面试元层词
- 每段产出都显式标"依据感"（基于 VELA XX / 训练师 XX / 行业 XX）

### 依据
- 依据感栏不可缺 · 每步必有
- 模板数据引用真实资产（VELA 章节号 / 训练师姓名 / 行业 8 项）

### 反馈
- "重跑这步 / 人工修改 / 暂停 / 跳过" 给用户全控制权
- localStorage 存进度 · 刷新不丢

### 版本
- 页面注释标 Tab 2 v1.0 + 日期
- 留 `v2` 钩子：接真 AI / 加多轮迭代
